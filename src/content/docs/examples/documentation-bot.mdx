# Documentation Bot Example

Build a technical documentation chatbot that helps developers understand your API, SDK, or platform by answering questions based on your technical documentation.

---

## Overview

A documentation bot helps developers:
- Find API endpoints and methods
- Understand code examples
- Troubleshoot integration issues
- Learn best practices

**Best for:** APIs, SDKs, developer tools, technical platforms

---

## Step 1: Create Your Bot

**Name:** `API Documentation Assistant` or `Developer Docs Bot`

**Description:** `Helps developers understand our API and integration guides`

**System Prompt:**
```
You are a technical documentation assistant. Your goal is to help 
developers understand our API, SDK, and platform.

Guidelines:
- Provide accurate technical information
- Include code examples when relevant
- Reference specific API versions
- Explain error messages clearly
- Suggest troubleshooting steps
- Link to relevant documentation sections when possible
- Use technical terminology appropriately
- Be precise and clear
```

---

## Step 2: Add Documentation

### Document 1: API Reference

**Title:** `API Reference`

**Content:**
```
# API Reference

## Authentication

All API requests require authentication using an API key:

```bash
Authorization: Bearer sk_live_your_api_key_here
```

## Endpoints

### POST /api/chat

Send a message to a chatbot.

**Request:**
```json
{
  "message": "Hello",
  "bot_slug": "my-bot",
  "stream": false
}
```

**Response:**
```json
{
  "response": "Hello! How can I help?",
  "conversationId": "uuid-here"
}
```

### GET /api/v1/documents

List all documents.

**Headers:**
- Authorization: Bearer {api_key}

**Response:**
```json
{
  "documents": [...]
}
```
```

### Document 2: Code Examples

**Title:** `Code Examples`

**Content:**
```
# Code Examples

## JavaScript/Node.js

```javascript
const response = await fetch('https://chatiq.io/api/chat', {
  method: 'POST',
  headers: {
    'Content-Type': 'application/json',
    'Authorization': 'Bearer sk_live_your_api_key',
  },
  body: JSON.stringify({
    message: 'Hello',
    bot_slug: 'my-bot',
  }),
});

const data = await response.json();
console.log(data.response);
```

## Python

```python
import requests

response = requests.post(
    'https://chatiq.io/api/chat',
    headers={
        'Authorization': 'Bearer sk_live_your_api_key',
    },
    json={
        'message': 'Hello',
        'bot_slug': 'my-bot',
    }
)

data = response.json()
print(data['response'])
```

## cURL

```bash
curl -X POST https://chatiq.io/api/chat \
  -H "Authorization: Bearer sk_live_your_api_key" \
  -H "Content-Type: application/json" \
  -d '{
    "message": "Hello",
    "bot_slug": "my-bot"
  }'
```

### Document 3: Error Codes

**Title:** `Error Codes Reference`

**Content:**
```
# Error Codes

## 401 Unauthorized

**Code:** `UNAUTHORIZED`

**Message:** Invalid or missing API key

**Solution:**
- Check that your API key is correct
- Ensure the Authorization header is formatted correctly
- Verify the API key is active in your dashboard

## 429 Too Many Requests

**Code:** `RATE_LIMIT`

**Message:** Rate limit exceeded

**Solution:**
- Check your plan's rate limits
- Implement exponential backoff
- Consider upgrading your plan

## 400 Bad Request

**Code:** `VALIDATION_ERROR`

**Message:** Invalid request parameters

**Solution:**
- Verify all required fields are present
- Check data types match the API specification
- Review the request body format
```
```

## Best Practices

- **Keep docs current** - Update when APIs change
- **Include code examples** - Show, don't just tell
- **Version your docs** - Reference specific API versions
- **Explain errors clearly** - Help developers debug
- **Link to resources** - Point to external docs when relevant
